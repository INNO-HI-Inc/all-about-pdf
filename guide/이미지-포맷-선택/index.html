JPG·PNG·WEBP·AVIF·SVG, 언제 어떤 포맷을 쓸까

사진은 JPG, 로고는 PNG나 SVG, 웹은 WEBP. 이미지 포맷마다 잘하는 일이 다릅니다. 변환할 때 무엇이 손실되는지까지 정리했습니다.

읽는 시간 약 8분·2026-07-20 업데이트·PDF의 모든 것 편집팀

이미지 파일을 열어 보면 확장자가 제각각입니다. jpg, png, webp, avif, svg… 이름만 다른 게 아니라 잘하는 일이 서로 다릅니다. 어떤 상황에 무엇을 골라야 하는지, 그리고 포맷을 바꿀 때 무엇이 사라지는지 정리했습니다.

01포맷을 가르는 두 가지 기준

포맷이 많아 보이지만, 실제로는 두 가지 질문만 던지면 대부분 정리됩니다.

  • 화질을 깎아서 용량을 줄이는가(손실), 원본 그대로 보관하는가(무손실) — JPG는 손실, PNG는 무손실입니다. WEBP와 AVIF는 두 방식을 모두 지원합니다.
  • 점으로 그려진 그림인가(래스터), 수식으로 그려진 그림인가(벡터) — JPG·PNG·WEBP·AVIF·GIF는 모두 픽셀을 격자에 채워 넣은 래스터입니다. SVG만 벡터입니다.

여기에 투명 배경을 지원하는지만 더하면 판단 근거는 거의 갖춰집니다. JPG는 투명을 지원하지 않고, PNG·WEBP·AVIF·SVG는 지원합니다. 이 세 가지 축을 기억해 두면 아래 내용이 훨씬 쉽게 읽힙니다.

02JPG — 사진을 위한 기본값

JPG(JPEG)는 사진에 최적화된 손실 압축 포맷입니다. 사람 눈이 잘 알아채지 못하는 미세한 색 변화를 버리는 방식으로 용량을 크게 줄입니다. 풍경, 인물, 음식처럼 색이 부드럽게 이어지는 이미지에서는 압축 흔적이 거의 보이지 않으면서 파일이 가벼워집니다.

반대로 경계선이 뚜렷한 이미지에는 약합니다. 글자가 들어간 스크린샷이나 단색 로고를 JPG로 저장하면 글자 테두리 주변이 뿌옇게 번지는 현상이 생깁니다. 압축 방식 자체가 급격한 색 변화를 다루기에 불리하기 때문입니다.

한 가지 더 주의할 점은 세대 손실입니다. JPG를 열어서 조금 고치고 다시 JPG로 저장하는 과정을 반복하면, 저장할 때마다 압축이 다시 적용되면서 화질이 조금씩 깎입니다. 한두 번은 티가 안 나지만 여러 번 반복하면 눈에 보이게 나빠집니다. 편집을 여러 차례 할 예정이라면 작업 중에는 PNG처럼 무손실 포맷으로 두고, 마지막에 JPG로 내보내는 편이 안전합니다.

03PNG — 투명과 선명함이 필요할 때

PNG는 무손실 포맷입니다. 저장해도 원본 픽셀이 그대로 보존되므로 몇 번을 다시 저장해도 화질이 나빠지지 않습니다. 여기에 투명 배경을 지원한다는 점이 결정적입니다. 배경이 비치는 로고, 아이콘, 도장 이미지는 PNG가 사실상 유일한 보편적 선택지입니다.

글자가 들어간 이미지에도 강합니다. 화면 캡처, 표나 차트 이미지, 발표 자료 슬라이드를 이미지로 저장할 때는 PNG가 글자 윤곽을 또렷하게 지켜 줍니다.

단점은 사진에서 용량이 커진다는 것입니다. 무손실이다 보니 사진처럼 색이 무수히 많은 이미지에서는 파일 크기가 JPG의 몇 배까지 늘어나기도 합니다. 카메라로 찍은 사진을 PNG로 보관할 이유는 거의 없습니다.

사이트에서는 JPG → PNG 변환, PNG → JPG 변환로 두 포맷 사이를 오갈 수 있습니다. 다만 방향에 따라 결과가 다르니 뒤의 '변환 방향' 항목을 꼭 확인하세요.

04WEBP와 AVIF — 더 가볍게 만드는 최신 포맷

WEBP는 손실·무손실을 모두 지원하고 투명 배경도 다룹니다. 같은 화질 기준으로 보면 대체로 JPG나 PNG보다 파일이 작아지는 경향이 있어, 웹사이트 로딩 속도를 줄이려는 목적에 잘 맞습니다. 요즘 웹에서 이미지를 저장하면 webp로 떨어지는 경우가 많은 것도 이 때문입니다.

AVIF는 여기서 한 걸음 더 나아간 포맷으로, 압축 효율이 가장 좋은 축에 듭니다. 같은 용량이면 화질이 더 낫고, 같은 화질이면 용량이 더 작습니다.

문제는 호환성입니다. 최신 브라우저에서는 대체로 잘 열리지만, 오래된 프로그램이나 사내 시스템, 일부 문서 편집기, 관공서 업로드 창에서는 webp나 avif를 아예 받아 주지 않는 경우가 있습니다. AVIF는 WEBP보다 지원 범위가 더 좁습니다. 그래서 내가 볼 용도라면 WEBP·AVIF, 남에게 보내거나 어딘가에 제출할 용도라면 JPG·PNG라는 기준이 실무에서 잘 통합니다.

받은 파일이 열리지 않을 때는 WEBP → JPG 변환, WEBP → PNG 변환, AVIF → JPG 변환, AVIF → PNG 변환로 익숙한 포맷으로 바꿔 쓰면 됩니다. 반대로 내 사이트에 올릴 이미지를 가볍게 만들고 싶다면 JPG → WEBP 변환, PNG → WEBP 변환를 쓰세요.

05GIF와 SVG — 용도가 뚜렷한 두 포맷

GIF는 색을 256개까지만 담을 수 있습니다. 그래서 짧은 애니메이션이나 단순한 도형에는 쓸 만하지만, 정지 사진을 GIF로 저장하면 색이 뭉개지고 얼룩덜룩해집니다. GIF로 받은 정지 이미지는 GIF → JPG 변환나 GIF → PNG 변환로 바꿔 두는 편이 다루기 편합니다. 다만 애니메이션 GIF를 정지 포맷으로 바꾸면 움직임은 사라집니다.

SVG는 성격이 완전히 다릅니다. 픽셀이 아니라 '어디서 어디까지 선을 긋고 어떤 색을 채워라'라는 도형 정보로 저장됩니다. 그래서 아무리 확대해도 계단처럼 깨지지 않습니다. 명함 크기로 쓰던 로고를 현수막 크기로 키워도 선이 그대로 매끈합니다. 회사 로고, 아이콘, 간단한 도표에 적합합니다.

대신 사진은 담을 수 없습니다. 사진은 수백만 개의 불규칙한 색점으로 이루어져 있어 도형으로 표현할 수 없기 때문입니다. 또 SVG를 받아 주지 않는 곳도 많아, 인쇄소나 문서에 넣으려면 SVG → PNG 변환(투명 유지), SVG → JPG 변환(흰 배경), SVG → WEBP 변환로 바꿔야 할 때가 있습니다. 이때 원본 SVG 파일은 반드시 따로 보관해 두세요. 한 번 래스터로 바꾸면 벡터로 되돌릴 수 없습니다.

06변환할 때 무엇이 사라지는지 — 방향이 중요합니다

포맷 변환에서 가장 많이 하는 오해는 '좋은 포맷으로 바꾸면 화질이 좋아진다'는 생각입니다. 그런 일은 일어나지 않습니다. 이미 사라진 정보는 어떤 포맷으로 바꿔도 돌아오지 않습니다.

  • JPG → PNG: 화질이 복구되지 않습니다. JPG 압축으로 이미 날아간 세부 정보를 무손실로 포장할 뿐이라, 용량만 크게 늘어납니다. 의미가 있는 경우는 '더 이상 화질이 깎이지 않게 고정하고 싶을 때'나 '투명 배경을 추가로 편집할 때'입니다.
  • PNG → JPG: 투명 영역이 사라집니다. 투명했던 부분은 보통 흰색으로 채워집니다. 어두운 배경에 올릴 로고를 이렇게 변환하면 흰 네모 상자가 그대로 드러납니다.
  • WEBP·AVIF → JPG/PNG: 호환성은 좋아지지만 용량은 대개 커집니다. 원본이 손실 압축이었다면 화질도 그만큼만 남습니다.
  • SVG → JPG/PNG/WEBP: 벡터의 장점을 잃습니다. 변환 시점의 크기로 픽셀이 고정되므로, 나중에 크게 쓰려면 원본 SVG에서 다시 큰 크기로 뽑아야 합니다.
  • GIF → JPG/PNG: 애니메이션이 사라지고, 256색으로 이미 줄어든 색은 되살아나지 않습니다.

정리하면 정보가 줄어드는 방향의 변환은 자유롭지만, 늘어나는 방향은 존재하지 않습니다. 원본은 언제나 따로 보관하는 습관이 가장 확실한 대비책입니다.

07상황별로 무엇을 고를까

  • 웹사이트에 올릴 사진 — WEBP가 1순위입니다. 방문자 환경이 걱정된다면 JPG로 두어도 무리 없습니다. AVIF는 속도를 최우선으로 하고 방문자 대부분이 최신 브라우저를 쓸 때 검토하세요.
  • 로고·아이콘 — 원본은 SVG로 보관하고, 필요할 때마다 필요한 크기의 PNG로 뽑아 쓰는 방식이 가장 낭비가 적습니다. 투명 배경이 필요하면 JPG는 후보에서 빼세요.
  • 문서 스캔본·화면 캡처 — 글자가 많으므로 PNG를 권합니다. 장수가 많아 용량이 부담이라면 JPG로 저장하되, 나중에 다시 편집하고 저장하는 일은 피하세요.
  • 카카오톡·메일 첨부 — 상대방 기기를 알 수 없으니 JPG나 PNG가 안전합니다. WEBP나 AVIF는 열리지 않는다는 연락을 받을 가능성이 있습니다.
  • 인쇄용 — 무손실 PNG, 로고라면 SVG 원본을 함께 넘기세요. 인쇄는 화면보다 훨씬 촘촘하게 찍히기 때문에 압축 흔적이 그대로 드러납니다. 웹에서 쓰던 작은 이미지를 확대해 인쇄하면 흐릿하게 나옵니다.
  • 오래된 프로그램에서 열어야 할 때 — JPG나 PNG로 변환하세요. 가장 널리 지원되는 두 포맷입니다.

08이미지와 PDF 사이

이미지 여러 장을 한 사람에게 보내야 할 때, 파일을 낱개로 보내면 순서가 뒤섞이거나 일부가 누락되기 쉽습니다. 이럴 때는 PDF 한 장으로 묶는 편이 확실합니다. 영수증 사진, 스캔한 계약서, 촬영한 과제물처럼 순서와 묶음이 중요한 자료에 특히 그렇습니다. 이미지 PDF 변환에서 이미지 여러 장을 원하는 순서로 하나의 PDF로 만들 수 있습니다.

반대로 PDF 안의 화면을 이미지로 꺼내야 할 때도 있습니다. 블로그에 도표를 올리거나, 발표 자료에 특정 페이지를 붙여 넣을 때가 그렇습니다. PDF→이미지 변환는 PDF의 각 페이지를 이미지 파일로 뽑아 줍니다.

참고로 이 사이트의 모든 변환은 브라우저 안에서만 처리됩니다. 이미지가 서버로 올라가지 않으므로 신분증 사본이나 계약서 스캔본처럼 민감한 이미지도 밖으로 나갈 걱정 없이 다룰 수 있습니다.

Q자주 묻는 질문

JPG를 PNG로 바꾸면 화질이 좋아지나요?

좋아지지 않습니다. JPG로 저장되는 과정에서 이미 버려진 정보는 어떤 포맷으로 바꿔도 되살아나지 않습니다. PNG로 바꾸면 '지금 상태 그대로' 무손실 보관되는 것이라, 화질은 그대로인데 용량만 커집니다. PNG 변환이 의미 있는 경우는 앞으로 여러 번 편집·저장할 예정이라 더 이상 화질이 깎이지 않게 고정하고 싶을 때입니다.

PNG를 JPG로 바꿨더니 투명 배경이 흰색이 됐어요.

정상 동작입니다. JPG는 투명을 저장할 수 없는 포맷이라 투명했던 영역을 어떤 색으로든 채워야 하고, 보통 흰색이 들어갑니다. 투명 배경을 유지해야 한다면 PNG나 WEBP를 쓰세요. 이미 JPG로 바꾼 뒤에는 흰 배경을 자동으로 되돌릴 수 없으니, PNG 원본이 있다면 그쪽을 다시 사용하시는 편이 빠릅니다.

WEBP 파일이 안 열립니다. 어떻게 해야 하나요?

최신 브라우저는 대부분 WEBP를 열지만, 오래된 이미지 뷰어나 일부 문서 편집기, 업로드 제한이 있는 사이트에서는 지원하지 않는 경우가 있습니다. 이럴 때는 <a href="/webp-to-jpg/">WEBP → JPG 변환</a>(사진용)나 <a href="/webp-to-png/">WEBP → PNG 변환</a>(투명 배경이 필요할 때)로 바꾸면 됩니다. AVIF도 마찬가지로 <a href="/avif-to-jpg/">AVIF → JPG 변환</a>, <a href="/avif-to-png/">AVIF → PNG 변환</a>를 쓰면 됩니다.

로고 파일은 어떤 포맷으로 보관해야 하나요?

가능하면 SVG 원본을 기준으로 보관하세요. 벡터라서 명함부터 현수막까지 어떤 크기로 키워도 깨지지 않습니다. 그리고 필요할 때마다 그 SVG에서 필요한 크기의 PNG를 뽑아 쓰면 됩니다. 반대로 PNG나 JPG만 남기고 SVG를 버리면 나중에 큰 크기가 필요할 때 다시 만들어야 하므로, 원본 SVG는 꼭 따로 챙겨 두시길 권합니다.
