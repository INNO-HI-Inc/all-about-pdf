PDF 합치기와 나누기, 실무에서 자주 막히는 지점

여러 장 스캔본 묶기, 표지·부록 붙이기, 필요한 조항만 발췌하기. PDF를 합치고 나눌 때 순서가 뒤섞이거나 방향이 제각각일 때 어떻게 푸는지 정리했습니다.

읽는 시간 약 6분·2026-07-20 업데이트

PDF를 합치고 나누는 일 자체는 어렵지 않습니다. 정작 시간을 잡아먹는 건 순서가 뒤바뀌고, 스캔본이 옆으로 누워 있고, 비밀번호가 걸려 있어 아예 열리지 않는 상황입니다. 실제로 자주 겪는 지점만 골라 정리했습니다.

01이럴 때 합치고, 이럴 때 나눕니다

기능 설명보다 상황부터 보는 편이 빠릅니다. 실무에서 합치기와 나누기가 필요한 순간은 대체로 다음과 같습니다.

  • 한 장씩 따로 저장된 스캔본 — 복합기나 스캐너 앱으로 계약서를 스캔하면 페이지마다 파일이 하나씩 생기는 경우가 많습니다. 12장이면 파일이 12개입니다. 이걸 그대로 메일에 첨부하면 받는 쪽이 순서를 짐작해가며 열어야 합니다.
  • 표지와 부록을 앞뒤로 붙일 때 — 본문은 워드, 표지는 파워포인트, 부록 표는 엑셀에서 각각 PDF로 저장한 뒤 하나로 묶는 흐름입니다. 제출용 보고서에서 가장 흔한 경우입니다.
  • 계약서에서 필요한 조항만 보낼 때 — 30페이지짜리 계약서 전체를 보내는 대신, 해당 조항이 있는 3페이지만 뽑아 보내는 편이 안전하고 상대도 편합니다.
  • 용량 제한에 걸려 반려된 첨부 — 메일이나 온라인 접수 시스템이 첨부 용량을 제한해 되돌아온 경우, 문서를 두세 덩어리로 나눠 순서대로 보냅니다.
  • 양면 스캐너가 앞면·뒷면을 따로 만든 경우 — 홀수 페이지 묶음과 짝수 페이지 묶음이 별개 파일로 나옵니다. 그냥 이어 붙이면 1,3,5 다음에 2,4,6이 오는 이상한 문서가 됩니다.

앞의 두 가지는 merge, 세 번째는 extract, 네 번째는 split의 영역입니다. 마지막 경우는 조금 더 손이 갑니다.

02합치기 전 5분 체크리스트

합친 다음에 문제를 발견하면 처음부터 다시 해야 합니다. 올리기 전에 네 가지만 확인하세요.

  • 순서가 내가 원하는 대로인가 — 파일 목록에 뜨는 순서와 실제 페이지 순서가 다를 수 있습니다. 아래에서 따로 다룹니다.
  • 비밀번호가 걸린 파일이 섞여 있지 않은가 — 열 때 암호를 묻는 PDF는 합치기 단계에서 그냥 실패합니다. 아는 비밀번호로 unlock에서 먼저 잠금을 푼 뒤 합쳐야 합니다.
  • 스캔본 방향이 제각각이지 않은가 — 스캔할 때 종이를 돌려 넣었거나 가로 문서가 섞이면, 합친 문서를 넘길 때마다 화면을 돌려야 합니다. rotate에서 미리 방향을 맞춰두면 깔끔합니다.
  • 빈 페이지가 끼어 있지 않은가 — 양면 스캔을 하면 뒷면이 백지인 장까지 그대로 들어옵니다. 합치기 전이든 후든 delete로 빼두는 편이 낫습니다.

여기까지 확인했다면 merge에서 파일을 올리고 순서를 잡은 뒤 합치면 됩니다.

031, 10, 2 — 파일명 정렬이 순서를 망치는 이유

가장 자주 나오는 사고입니다. 스캔 파일이 scan1.pdf, scan2.pdf … scan12.pdf처럼 저장되면, 컴퓨터는 이 이름을 숫자가 아니라 글자로 봅니다. 그래서 정렬하면 1, 10, 11, 12, 2, 3 순서가 됩니다. 이 상태로 한꺼번에 올려 합치면 페이지가 뒤죽박죽인 문서가 만들어집니다.

더 곤란한 건 눈에 잘 안 띈다는 점입니다. 스캔본은 페이지마다 생김새가 비슷해서, 합친 뒤 대충 넘겨보면 멀쩡해 보입니다. 상대방이 읽다가 말이 안 이어진다고 연락을 주고 나서야 알게 되는 경우가 많습니다.

해결책은 두 가지입니다.

  • 올리기 전에 파일명 자릿수를 맞춥니다. scan1 대신 scan001, scan002처럼 앞에 0을 채워 이름을 바꾸면 어떤 환경에서도 순서가 어긋나지 않습니다. 파일이 열 개를 넘어갈 때는 이 방법이 가장 확실합니다.
  • 합친 뒤 organize에서 눈으로 확인합니다. 페이지를 축소 화면으로 늘어놓고 보면 순서가 틀어진 곳이 바로 보입니다. 그 자리에서 끌어다 옮겨 고치면 되니, 다시 합칠 필요가 없습니다.

04페이지 크기가 섞였을 때 생기는 일

A4로 만든 본문에 Letter 규격 문서나 가로형 표를 붙이면, 합치는 것 자체는 문제없이 됩니다. 각 페이지는 원래 크기를 그대로 유지합니다. 문제는 그 다음입니다.

화면에서는 페이지마다 크기가 들쭉날쭉해 보이고, 인쇄하면 어떤 장은 여백이 넓고 어떤 장은 가장자리 글자가 잘립니다. 해외에서 받은 문서나 오래된 양식에 Letter 규격이 종종 섞여 있으니, 출력해서 제출할 서류라면 합치기 전에 각 파일의 페이지 크기를 확인해두세요. 문서 속성이나 인쇄 미리보기에서 볼 수 있습니다.

가로형 표나 도면처럼 원래 크기가 다를 수밖에 없는 페이지라면, 억지로 맞추기보다 rotate로 방향만 통일해두는 편이 읽기에 낫습니다. 인쇄용이 아니라 화면으로만 볼 문서라면 크기가 섞여 있어도 실무상 큰 문제는 없습니다.

05나눌 때는 범위 지정이 전부입니다

나누기에서 가장 많이 실수하는 지점은 페이지 번호를 어디 기준으로 세느냐입니다. 도구가 인식하는 번호는 종이에 인쇄된 쪽번호가 아니라 PDF의 물리적 순번입니다. 표지와 목차가 앞에 두 장 있으면, 문서에 '1페이지'라고 적힌 장은 실제로는 3번째 페이지입니다. 계약서에서 조항 페이지만 뽑을 때 이 차이 때문에 엉뚱한 장이 나오는 일이 흔합니다. 범위를 넣기 전에 뷰어에서 해당 장이 몇 번째로 표시되는지 먼저 확인하세요.

상황에 따라 쓰는 도구가 다릅니다.

  • 필요한 페이지가 몇 장뿐일 때 — extract에서 8-10처럼 범위를 지정해 그 부분만 새 파일로 뽑습니다. 계약서 조항 발췌가 여기에 해당합니다.
  • 뺄 페이지가 더 적을 때 — 50페이지 중 3장만 빼야 한다면 delete가 빠릅니다.
  • 문서를 통째로 두세 덩어리로 쪼갤 때 — split에서 자를 지점을 정하면 됩니다. 용량 초과로 반려된 첨부가 대표적입니다.

다만 용량 때문에 나눌 때는 기대치를 조정해야 합니다. 세 개로 나눠도 세 파일 용량의 합은 원본과 비슷합니다. 개당 제한을 피할 수는 있어도 총량이 줄지는 않습니다. 스캔 해상도가 높아 용량이 큰 문서라면, 나누기 전에 스캔 설정을 낮춰 다시 뜨는 편이 근본적인 해결입니다.

06양면 스캔이 홀수·짝수로 갈라졌을 때

단면 스캐너로 양면 문서를 처리할 때 흔히 쓰는 방법이 있습니다. 먼저 앞면을 전부 넣어 스캔하고, 종이 묶음을 뒤집어 뒷면을 다시 스캔하는 방식입니다. 이러면 파일이 두 개 나옵니다. 하나는 1, 3, 5… 홀수 페이지, 다른 하나는 짝수 페이지입니다.

여기서 함정은 뒷면 파일의 순서가 거꾸로일 때가 많다는 점입니다. 묶음을 통째로 뒤집어 넣으면 마지막 장의 뒷면부터 스캔되기 때문입니다. 즉 짝수 파일이 2, 4, 6 순서가 아니라 반대로 담겨 있습니다.

순서대로 처리하면 됩니다.

  • 짝수 파일을 먼저 열어 실제로 뒤집혀 있는지 확인합니다. 뒤집혀 있다면 reverse로 페이지 순서를 되돌립니다.
  • 두 파일을 merge로 합칩니다. 이 시점에는 홀수 묶음 뒤에 짝수 묶음이 붙은 상태입니다.
  • organize에서 페이지를 끌어다 1, 2, 3… 순서로 끼워 넣습니다. 축소 화면으로 보면서 옮기니 어느 장이 어디 들어갈지 헷갈리지 않습니다.

마지막으로 합친 문서에 쪽번호가 없다면 page-numbers로 넣어두세요. 제출한 뒤 상대와 통화하며 '몇 페이지 보세요'라고 말할 수 있는지 없는지는 생각보다 큰 차이입니다.

참고로 이 사이트의 도구는 모두 브라우저 안에서만 동작합니다. 계약서나 신분증 스캔본처럼 밖으로 나가면 곤란한 문서도 파일이 서버로 올라가지 않으니, 업로드를 꺼려 미뤄두던 작업에 쓰기 좋습니다.

Q자주 묻는 질문

합칠 파일이 20개가 넘는데 순서를 하나하나 맞춰야 하나요?

올리기 전에 파일명을 001, 002처럼 자릿수를 맞춰 바꿔두는 편이 훨씬 빠릅니다. 그러면 이름 순서와 페이지 순서가 일치해서 그대로 합치면 됩니다. 이름을 바꾸기 곤란하다면 일단 합친 뒤 organize에서 틀어진 부분만 끌어다 고치는 방법도 있습니다.

비밀번호가 걸린 PDF도 바로 합칠 수 있나요?

아니요, 잠금을 먼저 풀어야 합니다. 열 때 암호를 묻는 PDF는 내용을 읽을 수 없어 합치기 단계에서 실패합니다. 아는 비밀번호를 unlock에 입력해 잠금을 해제한 파일로 합쳐 주세요. 비밀번호를 모르는 문서는 풀 수 없으며, 이 도구는 암호를 알아내는 용도가 아닙니다.

PDF를 나누면 용량이 줄어드나요?

나눈 파일들의 용량을 다 더하면 원본과 비슷합니다. 파일 하나당 용량 제한을 피하는 데는 효과가 있지만 전체 용량이 줄지는 않습니다. 스캔본이라 용량이 크다면 스캔 해상도를 낮춰 다시 뜨는 편이 근본적인 해결책입니다.

계약서에서 특정 조항 페이지만 뽑았는데 다른 내용이 나옵니다.

종이에 인쇄된 쪽번호와 PDF의 실제 페이지 순번이 어긋나서 생기는 일입니다. 표지나 목차가 앞에 붙어 있으면 문서에 적힌 1페이지가 실제로는 3번째 페이지입니다. 뷰어에서 그 장이 몇 번째로 표시되는지 확인한 뒤 extract에 그 번호를 넣으세요.
