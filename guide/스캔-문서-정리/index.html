스캔한 문서를 깔끔한 PDF로 정리하는 법

돌아간 페이지, 빈 뒷면, 검은 테두리, 뒤집힌 순서, 큰 용량까지. 스캔본 특유의 문제를 순서대로 잡는 방법과 각 단계에 쓰는 도구를 정리했습니다.

읽는 시간 약 8분·2026-07-20 업데이트·PDF의 모든 것 편집팀

스캐너에서 막 나온 PDF는 대체로 그대로 쓰기 어렵습니다. 페이지가 옆으로 누워 있고, 빈 뒷면이 끼어 있고, 가장자리에 검은 테두리가 남고, 용량은 수십 메가입니다. 문제를 하나씩 잡되 순서를 지켜야 두 번 일하지 않습니다.

01스캔본이 지저분해 보이는 진짜 이유

스캔 PDF가 어수선한 건 대부분 같은 원인 몇 가지에서 나옵니다. 원인을 먼저 구분해 두면 어떤 도구를 써야 할지가 자연스럽게 정해집니다.

  • 페이지 방향이 어긋남 — 가로로 된 표를 세로 급지로 넣으면 90도로 눕고, 원고를 거꾸로 놓으면 180도로 뒤집힙니다. 화면에서 고개를 돌려 읽어야 하는 상태입니다.
  • 빈 뒷면이 같이 들어옴 — 자동급지(ADF)에서 양면 스캔을 켜 두면, 뒷면이 백지인 문서도 빈 페이지 한 장으로 저장됩니다. 20장짜리 단면 문서가 40페이지가 되는 이유입니다.
  • 가장자리에 검은 테두리나 그림자 — 원본보다 넓은 유리면에 올려두면 덮개 밖의 어두운 영역이 그대로 찍힙니다. 책을 스캔하면 접힌 부분에 짙은 그림자가 남습니다.
  • 페이지 순서가 뒤집힘 — 급지 방향을 반대로 넣으면 마지막 장이 1페이지가 됩니다. 계약서나 보고서에서는 그대로 제출하면 곤란한 실수입니다.
  • 용량이 지나치게 큼 — 스캔본은 페이지마다 사진 한 장이 통째로 들어 있습니다. 컬러 600dpi로 뜬 30페이지면 100MB를 넘기기도 합니다. 메일 첨부나 제출 사이트 용량 제한에 자주 걸립니다.
  • 페이지 크기가 제각각 — A4 문서 사이에 영수증이나 명함을 섞어 넣으면 페이지마다 크기가 달라져, 인쇄할 때 배율이 들쭉날쭉해집니다.

02정리하기 전에 알아둘 것: 스캔본은 문서가 아니라 사진입니다

이 점을 먼저 짚고 가야 헛수고를 피할 수 있습니다. 스캐너가 만드는 PDF는 종이를 찍은 사진을 페이지 크기에 맞춰 담아 놓은 파일입니다. 사람 눈에는 글자로 보이지만, 컴퓨터 입장에서는 글자가 아니라 점의 배열입니다. 그래서 본문을 드래그해도 선택되지 않고, Ctrl+F로 분명히 있는 단어를 검색해도 찾지 못합니다.

내가 가진 PDF가 사진인지 텍스트인지는 30초면 확인됩니다. PDF를 열어 본문 아무 줄이나 드래그해 보세요. 파란 선택 영역이 글자 모양대로 잡히면 텍스트가 들어 있는 PDF이고, 아무것도 잡히지 않거나 페이지 전체가 하나의 그림처럼 잡히면 스캔 이미지입니다.

이걸 진짜 글자로 바꾸려면 OCR(광학 문자 인식)이 필요합니다. 솔직히 말씀드리면 이 사이트는 OCR을 제공하지 않습니다. extract-text(텍스트 추출)는 이미 텍스트가 들어 있는 PDF에서 그 글자를 뽑아내는 도구라, 스캔본에 돌리면 결과가 비거나 머리글 같은 부스러기만 나옵니다. 도구가 고장 난 게 아니라 애초에 뽑을 글자가 없는 것입니다. 검색 가능한 문서가 꼭 필요하다면 OCR 기능이 있는 별도 프로그램을 쓰셔야 합니다. 아래 정리 과정은 보기 좋고 다루기 편한 스캔본을 만드는 작업이지, 검색되는 문서로 바꾸는 작업이 아닙니다.

03순서를 지켜야 두 번 일하지 않습니다

권장 순서는 이렇습니다. 회전 바로잡기 → 페이지 순서 정리 → 빈 페이지 제거 → 필요 없는 페이지 삭제 → 여백 자르기 → (필요시) 흑백 변환 → 압축 → 페이지 번호. 그냥 나열한 게 아니라, 각 단계마다 뒤로 미루면 손해가 나는 이유가 있습니다.

  • 회전이 맨 앞인 이유 — 페이지가 누워 있으면 어디가 위쪽 여백인지 눈으로 판단할 수 없습니다. 방향이 틀어진 채로 자르기를 하면 엉뚱한 변이 잘려 나갑니다. 빈 페이지를 눈으로 확인하는 일도 방향이 맞아야 편합니다.
  • 빈 페이지·불필요 페이지를 자르기보다 먼저 — 어차피 버릴 페이지를 다듬는 데 시간을 쓸 이유가 없습니다. 페이지 수를 먼저 줄여 놓으면 이후 압축도 눈에 띄게 빨라집니다.
  • 자르기를 압축보다 먼저 — 압축은 페이지를 이미지로 다시 그리는 방식입니다. 검은 테두리를 미리 잘라내면 다시 그릴 면적 자체가 줄어 결과 파일이 더 가벼워집니다. 순서를 바꾸면 필요 없는 여백까지 정성껏 압축해 넣게 됩니다.
  • 흑백 변환과 압축은 각각 한 번씩만 — 두 작업 모두 페이지를 이미지로 재작성합니다. 흑백으로 바꾼 뒤 다시 압축하면 화질이 두 번 깎입니다. 둘 다 필요하면 흑백 → 압축 순으로 한 번씩만 거치고, 압축 화질은 '보통'부터 시도해 보세요.
  • 페이지 번호가 맨 마지막인 이유 — 두 가지입니다. 첫째, 페이지를 지우기 전에 번호를 매기면 삭제 후 번호가 건너뛰어 1, 2, 5, 6처럼 어긋납니다. 둘째, 번호를 먼저 넣고 압축하면 또렷하게 얹은 숫자까지 이미지로 뭉개집니다. 페이지 구성이 완전히 확정된 뒤 맨 마지막에 얹으세요.

04단계별로 실제 해보기

각 단계에 쓰는 도구입니다. 필요 없는 단계는 건너뛰어도 됩니다.

  • 1. 방향 바로잡기 — 전체 페이지가 똑같이 돌아갔다면 rotate에서 왼쪽 90도·오른쪽 90도·180도 중 하나를 골라 한 번에 처리합니다. 문제는 페이지마다 방향이 다른 경우인데, 이때는 organize를 쓰세요. 모든 페이지가 썸네일로 펼쳐지고 각 페이지를 개별로 돌릴 수 있습니다.
  • 2. 순서 정리 — 급지를 반대로 넣어 통째로 거꾸로 들어갔다면 reverse 한 번이면 끝납니다. 일부만 엉켰다면 organize에서 썸네일을 끌어다 옮기는 편이 확실합니다.
  • 3. 빈 페이지 제거 — remove-blank가 거의 흰 페이지만 자동으로 골라냅니다. 다만 스캔본은 종이 얼룩이나 뒷면 비침 때문에 완전한 백지가 아닌 경우가 있으니, 미리보기로 표시된 결과를 꼭 한 번 눈으로 확인하세요. 도장이나 손글씨 서명만 있는 페이지가 잘못 걸리지 않았는지 보는 것이 특히 중요합니다.
  • 4. 필요 없는 페이지 삭제 — 표지 뒷장, 잘못 스캔해 두 번 들어간 장, 안내문 같은 것은 delete에서 페이지 번호로 지웁니다.
  • 5. 여백 자르기 — crop에서 사방을 적게·보통·많이 중에 골라 잘라냅니다. 검은 테두리는 대개 '보통'이면 정리됩니다. 미리보기로 본문 글자가 잘리지 않았는지 확인하고, 과하게 잘렸다면 한 단계 낮추세요.
  • 6. 흑백 변환 — 도장이나 형광펜 색을 살릴 필요가 없다면 grayscale로 회색조로 바꿉니다. 색 정보가 빠지면서 용량이 줄고 흑백 프린터 출력도 깔끔해집니다. 컬러가 의미를 갖는 문서라면 이 단계는 건너뛰세요.
  • 7. 압축 — compress에서 화질을 골라 용량을 줄입니다. 스캔본은 원래 이미지라, 페이지를 다시 그리는 압축 방식과 궁합이 좋아 효과가 큰 편입니다. '보통'으로 먼저 해보고 용량이 여전히 크면 '낮음'을 시도하되, 작은 글씨가 뭉개지지 않았는지 확인하세요.
  • 8. 페이지 번호 — 제출용이라면 page-numbers로 마지막에 번호를 얹습니다.

자르기를 세게 해서 글자가 가장자리에 바짝 붙었거나 제본할 문서라면, 맨 뒤에 add-margin으로 흰 여백을 다시 넉넉히 더해 주면 인쇄할 때 글자가 잘리지 않습니다.

05스캔할 때 미리 잘하면 정리할 일이 반으로 줍니다

가장 확실한 정리는 애초에 잘 스캔하는 것입니다. 스캐너나 복합기 설정에서 세 가지만 챙기면 위 작업 대부분이 필요 없어집니다.

  • 해상도는 200~300dpi — 일반 문서는 이 정도면 충분히 또렷하게 읽히고 인쇄도 깨끗합니다. 600dpi는 사진이나 도면처럼 세밀한 원본에나 의미가 있고, 글자 문서에서는 파일만 몇 배로 무거워집니다.
  • 색이 필요 없으면 흑백이나 회색조로 — 계약서, 과제, 민원 서류처럼 검은 글자만 있는 문서를 컬러로 뜰 이유가 없습니다. 스캔 단계에서 회색조를 고르면 흑백 변환 단계를 통째로 건너뛸 수 있고, 나중에 변환하는 것보다 화질도 좋습니다.
  • 단면 문서는 양면 스캔을 끄고, 급지 방향을 확인 — 이 두 가지만 지켜도 빈 페이지 제거와 순서 뒤집기를 할 일이 없습니다. 원고를 넣기 전에 첫 장이 위로, 인쇄면이 바른 방향인지 한 번만 보면 됩니다.

페이지 크기가 섞이는 문제도 스캔 단계에서 막는 편이 낫습니다. 크기가 다른 종이는 따로 스캔해 나중에 합치고, 그래도 섞였다면 인쇄할 때 프린터 설정에서 '용지 크기에 맞춤'을 켜면 대체로 해결됩니다.

06이 도구들로는 해결되지 않는 것

과장하지 않고 한계도 적어 둡니다.

  • OCR(글자 인식) — 앞서 말씀드린 대로 제공하지 않습니다. 정리한 결과물도 여전히 이미지 기반 PDF이고, 검색과 복사는 되지 않습니다.
  • 기울기 자동 보정 — 종이가 비스듬히 들어가 3도쯤 기운 페이지를 반듯하게 펴 주는 기능은 없습니다. 회전은 90도 단위입니다. 기울어짐이 심하면 그 장만 다시 스캔하는 편이 빠릅니다.
  • 페이지별로 다르게 자르기 — 자르기는 전체 페이지에 같은 비율로 적용됩니다. 페이지마다 테두리 위치가 크게 다르면, 가장 안전한 정도로만 잘라내고 나머지는 남겨 두는 편이 낫습니다.

모든 작업은 파일을 서버로 보내지 않고 브라우저 안에서만 처리됩니다. 계약서나 신분증 사본처럼 밖으로 내보내기 꺼려지는 스캔본도 업로드 걱정 없이 정리할 수 있습니다.

Q자주 묻는 질문

스캔한 PDF에서 글자 복사와 검색이 안 됩니다. 어떻게 하나요?

스캔본은 종이를 찍은 이미지라서 컴퓨터가 글자로 인식하지 못하기 때문입니다. 진짜 텍스트로 바꾸려면 OCR(광학 문자 인식)이 필요한데, 이 사이트는 OCR을 제공하지 않습니다. <a href="/extract-text/">텍스트 추출</a>는 이미 텍스트가 들어 있는 PDF에서만 글자를 뽑아내므로 스캔본에 쓰면 결과가 비어 나옵니다. 검색 가능한 문서가 필요하시면 OCR 기능이 있는 프로그램을 따로 이용하세요.

페이지마다 돌아간 방향이 다른데 rotate로 한 번에 되나요?

<a href="/rotate/">회전</a>는 모든 페이지를 같은 방향으로 한 번에 돌리는 도구라, 페이지마다 방향이 다르면 맞지 않습니다. 이럴 때는 <a href="/organize/">페이지 정리</a>를 쓰세요. 전체 페이지가 썸네일로 펼쳐지고 페이지 하나하나를 개별로 회전할 수 있어서, 누운 페이지와 뒤집힌 페이지가 섞여 있어도 한 화면에서 정리됩니다.

스캔 PDF 용량은 얼마나 줄어드나요?

원본 상태에 따라 달라서 딱 잘라 말씀드리기 어렵습니다. 다만 줄이는 폭을 키우는 방법은 분명합니다. 빈 페이지와 필요 없는 페이지를 먼저 지우고, 검은 테두리를 잘라내고, 색이 필요 없으면 회색조로 바꾼 뒤 마지막에 압축하면 각 단계의 효과가 겹쳐 쌓입니다. 압축 화질은 '보통'부터 시도하고, 결과를 열어 작은 글씨가 읽히는지 확인한 다음 더 낮출지 정하세요.

정리 순서를 꼭 지켜야 하나요?

필요 없는 단계는 건너뛰어도 되지만, 두 가지는 지키시는 편이 좋습니다. 첫째, 페이지 번호는 반드시 맨 마지막에 넣으세요. 페이지를 지우기 전에 번호를 매기면 번호가 건너뛰어 어긋납니다. 둘째, 압축은 다른 편집을 모두 끝낸 뒤 한 번만 하세요. 압축은 페이지를 이미지로 다시 그리기 때문에 여러 번 반복하면 화질이 계속 깎입니다.

목차

  1. 스캔본이 지저분해 보이는 진짜 이유
  2. 정리하기 전에 알아둘 것: 스캔본은 문서가 아니라 사진입니다
  3. 순서를 지켜야 두 번 일하지 않습니다
  4. 단계별로 실제 해보기
  5. 스캔할 때 미리 잘하면 정리할 일이 반으로 줍니다
  6. 이 도구들로는 해결되지 않는 것
