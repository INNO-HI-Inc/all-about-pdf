PDF에 남는 보이지 않는 정보 - 메타데이터와 개인정보

이력서나 계약서 PDF에는 작성자 이름, 회사명, 원본 파일 경로가 함께 저장됩니다. 무엇이 남는지 확인하고 지우는 방법, 그리고 검은색으로 가려도 안 지워지는 이유를 정리했습니다.

읽는 시간 약 7분·2026-07-20 업데이트·PDF의 모든 것 편집팀

PDF를 열면 눈에 보이는 것은 본문뿐이지만, 파일 안에는 작성자 이름과 만든 프로그램, 원본 파일 경로 같은 정보가 함께 저장됩니다. 이력서나 제안서를 그대로 보낼 때 무엇이 같이 가는지, 어떻게 확인하고 지우는지 차분히 짚어보겠습니다.

01PDF 안에는 본문 말고도 무엇이 들어가나

워드나 한글, 파워포인트에서 '다른 이름으로 저장 - PDF'를 누르는 순간, 프로그램은 우리가 입력한 적 없는 항목까지 알아서 채워 넣습니다. PDF 규격에는 문서 정보를 담는 자리가 따로 마련되어 있고, 변환 프로그램이 그 자리를 비워두지 않기 때문입니다.

  • 작성자 — 컴퓨터에 계정을 만들 때 입력한 이름이나 회사에서 등록한 사용자명이 그대로 들어갑니다. 본인이 입력한 기억이 없어도 대부분 채워져 있습니다.
  • 제목, 주제, 키워드 — 직접 적지 않았더라도 원본 문서의 첫 줄이나 파일명이 자동으로 들어가는 경우가 있습니다.
  • 만든 프로그램과 변환 엔진 — 어떤 워드프로세서 몇 버전으로 작성했고, 어떤 변환기를 거쳐 PDF가 되었는지가 남습니다.
  • 생성 시각과 마지막 수정 시각 — 초 단위로, 시간대까지 함께 기록됩니다.
  • 회사명이나 부서 — 오피스 프로그램에 조직 정보가 등록되어 있으면 문서로 따라 넘어갑니다.
  • 원본 파일 경로 — 문서 안의 링크나 내부 정보에 C:\Users\사용자계정\바탕화면\... 같은 전체 경로가 남는 일이 있습니다. 계정명이 실명이나 사번인 경우가 많다는 점이 문제입니다.

이 정보들은 파일을 메일에 첨부하거나 채용 사이트에 올릴 때 본문과 함께 그대로 이동합니다. 받는 쪽에서 특별한 도구가 필요한 것도 아닙니다. PDF 뷰어의 '문서 속성' 메뉴 하나만 열면 대부분 보입니다.

02실제로 곤란해지는 상황들

이론적인 이야기가 아니라, 문서를 자주 주고받는 자리에서 반복적으로 생기는 일들입니다.

  • 이력서에 남는 이전 직장의 흔적 — 전 회사에서 쓰던 문서 템플릿을 고쳐 이력서를 만들면, 작성자나 회사명 항목에 이전 회사 이름이 그대로 남아 있는 경우가 있습니다. 본문은 깨끗하게 정리했는데 속성만 예전 그대로인 상태입니다.
  • 여러 곳에 보내는 제안서와 견적서 — 한 고객사용으로 만든 파일을 복사해 다른 고객사에 보낼 때가 많습니다. 이때 원본 파일 경로에 제안서\A사_최종\ 같은 폴더 이름이 남아 있으면, 받는 쪽에서 다른 거래처 이름을 보게 됩니다. 본문에서 회사명을 아무리 꼼꼼히 바꿔도 경로는 따로 바뀌지 않습니다.
  • 익명 제보와 블라인드 심사 — 이름을 밝히지 않는 것이 전제인 문서인데 작성자 항목에 실명이 박혀 있으면 익명이 아닙니다. 블라인드로 진행하는 공모전, 논문 심사, 사내 제보 문서에서 특히 조심해야 합니다.
  • 수정 이력이 드러나는 경우 — 마감 시각 이후에 만들어진 문서라는 사실이 생성 시각에 남아 있으면 설명이 필요해집니다. 반대로 오래전에 만든 문서를 새로 작성한 것처럼 보내는 경우도 마찬가지입니다.

공통점은 본문만 신경 쓰고 속성은 보지 않았다는 것입니다. 우리가 화면에서 확인하는 것은 본문뿐이니 자연스러운 일입니다.

03메타데이터를 지워도 남는 것 — 가장 중요한 부분

여기서부터가 오해가 많은 지점입니다. 메타데이터를 지우는 것과 문서 내용을 감추는 것은 전혀 다른 작업입니다.

첫째, 속성을 비워도 본문에 적힌 내용은 그대로 있습니다. 계약서에 상대방 이름과 계좌번호가 적혀 있다면 속성을 아무리 깨끗이 지워도 그 글자는 그 자리에 남아 있습니다. 메타데이터 제거는 문서를 익명으로 만들어주는 도구가 아니라, 문서에 딸려 있는 꼬리표를 떼는 도구입니다.

둘째, 검은색으로 덧칠해 가린 부분은 지워진 것이 아닙니다. 주민등록번호나 계좌번호 위에 검은 사각형을 그려 넣고 PDF로 저장하는 방식이 널리 쓰입니다. 하지만 PDF에서 도형은 글자 위에 겹쳐 놓인 별개의 요소일 뿐입니다. 아래에 있던 글자는 문서 안에 그대로 남아 있습니다. 그 부분을 마우스로 긁어 복사하면 검은 박스 아래에 있던 글자가 그대로 붙여넣기 되는 경우가 있고, 텍스트 추출 기능을 쓰면 더 쉽게 드러납니다. 실제로 공개된 문서에서 이 방식 때문에 가려야 할 내용이 노출된 사례가 국내외에서 여러 차례 있었습니다.

제대로 가리려면 덧칠이 아니라 해당 글자 자체를 없애야 합니다. 현실적인 방법은 두 가지입니다. 원본 문서에서 그 부분을 삭제하거나 다른 글자로 바꾼 뒤 다시 PDF로 만드는 것, 또는 가린 상태의 페이지를 이미지로 바꿔서 텍스트 계층 자체를 없애는 것입니다. 이미지로 바꾸면 검은 박스는 진짜 검은 픽셀이 되고, 그 아래에 있던 글자 정보는 함께 사라집니다.

04확인하고 지우는 순서

무작정 지우기 전에 무엇이 들어 있는지부터 보는 편이 낫습니다. 실제로 열어보면 아무것도 없는 문서도 있고, 예상 못 한 항목이 채워져 있는 문서도 있습니다.

  • 1단계, 확인 — pdf-info에 파일을 올리면 제목, 작성자, 만든 프로그램, 생성 시각 같은 항목과 페이지 수, 용량, 잠김 여부를 표로 보여줍니다. 지금 보내려는 파일에 무엇이 붙어 있는지 30초면 파악됩니다.
  • 2단계, 제거 — 지워야 할 항목이 있다면 remove-metadata로 문서 속성을 비웁니다. 본문과 페이지 구성은 건드리지 않고 속성만 비우기 때문에 문서가 깨지지 않습니다.
  • 3단계, 양식 입력값 정리 — 신청서나 서식처럼 입력 칸이 있는 PDF는 우리가 적은 값이 본문이 아니라 입력 필드에 따로 저장됩니다. 이 상태로는 나중에 값이 바뀌거나, 뷰어에 따라 다르게 보일 수 있습니다. flatten으로 평탄화하면 입력값이 페이지에 그대로 고정되고 입력 칸은 사라집니다.
  • 4단계, 최후의 수단 — 그래도 불안하거나 가림 처리한 부분이 있다면 to-image로 페이지를 이미지로 바꾸고, image-to-pdf로 다시 한 파일로 묶습니다. 텍스트 계층이 사라지므로 숨은 글자가 남지 않습니다. 대신 문서 안에서 글자 검색이나 복사가 안 되고 파일 용량이 눈에 띄게 커집니다. 필요한 경우에만 쓰는 방법입니다.

이미지로 다시 묶은 파일에도 새 속성이 생기므로, 마지막에 remove-metadata를 한 번 더 돌리고 pdf-info로 비었는지 확인하면 깔끔합니다.

05보내기 전 마지막 점검

중요한 문서를 내보내기 직전에 아래 항목만 훑어도 대부분의 사고는 막을 수 있습니다.

  • 작성자와 회사명 항목에 지금 상황과 맞지 않는 이름이 남아 있지 않은지
  • 제목 항목에 원본 파일명이나 폴더 이름이 들어가 있지 않은지
  • 가려야 할 부분을 검은 도형으로 덮어두기만 한 것은 아닌지. 그 부분을 긁어서 복사해보면 바로 알 수 있습니다
  • 입력 칸이 있는 서식이라면 값이 페이지에 고정되어 있는지
  • 파일명 자체에 실명이나 사번, 내부 프로젝트명이 들어 있지 않은지. 속성은 지워도 파일명은 그대로 전달됩니다

배포 범위를 제한하고 싶다면 별도의 도구도 있습니다. protect로 열기 비밀번호를 걸면 받는 사람만 열 수 있고, watermark로 '외부 유출 금지' 같은 표시를 넣으면 재배포 시 출처가 드러납니다. 다만 이 둘은 내용을 감추는 기능이 아니라 사용 조건을 붙이는 기능이라는 점은 구분해두는 편이 좋습니다.

이 사이트의 도구는 모두 브라우저 안에서만 동작합니다. 이력서나 계약서처럼 밖으로 나가면 곤란한 파일도 서버로 올라가지 않고 내 기기에서 처리되므로, 확인 단계에서 마음 놓고 열어볼 수 있습니다.

Q자주 묻는 질문

메타데이터를 지우면 문서가 깨지거나 인쇄가 달라지나요?

아니요. 제거되는 것은 제목, 작성자, 만든 프로그램 같은 문서 속성이고 본문 글자, 이미지, 페이지 순서, 여백은 그대로 유지됩니다. 화면에 보이는 모습과 인쇄 결과는 처리 전후가 동일합니다. 다만 문서 안에서 특정 속성을 참조하도록 만든 특수한 파일이라면 저장 후 <a href="/pdf-info/">정보 보기</a>로 한 번 확인해보는 것이 안전합니다.

검은색으로 가린 부분이 정말 읽히는지 어떻게 확인하나요?

가린 부분을 마우스로 드래그해서 선택한 다음 복사하고, 메모장 같은 곳에 붙여넣어 보세요. 아래에 있던 글자가 그대로 나오면 지워지지 않은 것입니다. 선택 자체가 안 되면 그 부분이 원래 이미지였을 가능성이 있지만, 그렇다고 안전하다고 단정하기는 어렵습니다. 확실하게 하려면 원본에서 해당 내용을 삭제하고 다시 PDF로 만들거나, <a href="/to-image/">PDF→이미지 변환</a>로 이미지화한 뒤 <a href="/image-to-pdf/">이미지 PDF 변환</a>로 묶는 방식을 쓰는 편이 낫습니다.

이미지로 바꾸면 완전히 안전한가요?

텍스트 계층이 사라지므로 숨어 있던 글자를 복사해서 읽어내는 방식은 막힙니다. 하지만 화면에 그대로 보이는 내용은 여전히 읽을 수 있고, 문자 인식 기능을 쓰면 이미지 속 글자도 텍스트로 옮길 수 있습니다. 즉 이미지화는 '숨은 정보'를 없애는 방법이지 '보이는 정보'를 감추는 방법이 아닙니다. 가려야 할 내용은 이미지로 바꾸기 전에 확실히 덮거나 삭제해두어야 합니다. 또 이미지로 만든 파일에도 새 속성이 생기므로 마지막에 <a href="/remove-metadata/">개인정보 제거</a>를 한 번 더 적용하는 것을 권합니다.

애초에 이런 정보가 안 들어가게 저장할 수는 없나요?

완전히 막기는 어렵습니다. 대부분의 워드프로세서는 PDF로 내보낼 때 작성자와 변환 프로그램 정보를 자동으로 넣고, 이를 끄는 설정이 없거나 눈에 잘 띄지 않는 곳에 있습니다. 프로그램의 사용자 이름을 미리 바꿔두면 작성자 항목은 어느 정도 관리할 수 있지만 생성 시각이나 변환 엔진 정보까지 없앨 수는 없습니다. 결국 만든 다음에 <a href="/pdf-info/">정보 보기</a>로 확인하고 <a href="/remove-metadata/">개인정보 제거</a>로 정리하는 절차를 습관으로 두는 편이 현실적입니다.
