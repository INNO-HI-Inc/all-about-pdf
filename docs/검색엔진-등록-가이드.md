# 검색엔진 등록 가이드 (네이버 + 구글)

「PDF의 모든 것」을 네이버·구글 검색에 노출시키기 위한 실전 절차입니다.
이 사이트는 검색 노출을 고려해 설계됐습니다(기능별 정적 페이지, 메타·OG·JSON-LD, sitemap, 빠른 로딩). 아래는 **배포 후** 진행하는 등록 작업입니다.

> 핵심 사실: **네이버 검색봇(Yeti)은 자바스크립트를 거의 렌더링하지 못합니다.** 그래서 이 사이트는 제목·설명·본문을 정적 HTML에 담고, JS는 도구 동작만 맡도록 만들었습니다. 이 구조 덕분에 네이버·구글 모두 색인 기반이 확보됩니다.

---

## 0. 배포 주소 확정 (가장 먼저)

1. `build.mjs` 상단의 `SITE_URL`을 실제 배포 주소로 바꿉니다.
   - 예: GitHub Pages 프로젝트 페이지 → `https://<아이디>.github.io/all-about-pdf`
   - 커스텀 도메인을 붙이면 → `https://example.com`
2. 터미널에서 `node build.mjs` 실행 → 모든 페이지의 canonical·OG·sitemap이 새 주소로 갱신됩니다.
3. 커밋·푸시 → 자동 배포.

> 커스텀 도메인을 쓰면 SEO상 더 유리합니다(서브경로 없는 루트 도메인). GitHub Pages는 Settings → Pages → Custom domain에서 연결하고, 저장소 루트에 `CNAME` 파일(도메인만 한 줄)을 둡니다.

## 1. 네이버 서치어드바이저 (웹마스터도구)

주소: https://searchadvisor.naver.com/

1. **로그인 → 웹마스터 도구 → 사이트 등록**에 배포 주소 입력
2. **소유확인** — 두 방법 중 "HTML 태그"가 가장 쉽습니다.
   - 발급된 `<meta name="naver-site-verification" content="...">` 값을 복사
   - 이 저장소의 각 페이지 `<head>`에 해당 주석이 이미 있습니다. **`build.mjs`의 `page()` 함수에서 주석을 풀고 content 값을 넣은 뒤** `node build.mjs` → 푸시
   - (또는 홈 `index.html`의 해당 주석만 풀어도 소유확인은 됩니다)
3. **요청 → 사이트맵 제출**: `sitemap.xml` 입력
4. **요청 → 웹 페이지 수집**: 홈과 주요 도구 URL을 직접 수집 요청(색인 촉진)
5. (선택) **RSS 제출**: 최신/중요 콘텐츠 알림용

> 네이버는 등록 후 사이트 정보를 수집·반영하는 데 통상 **약 2주** 걸리고, 그 뒤에도 모든 페이지 노출이 보장되진 않습니다. 본문 품질·방문 데이터가 쌓이며 점진적으로 올라갑니다.

### 네이버에서 더 잘 노출되려면
- 각 도구 페이지의 **본문 텍스트**(사용법·FAQ·활용예)가 충분해야 합니다 → 이미 700~1,000자 수준으로 작성됨
- `title`(28자 내외) / `description`(80자 이내)이 핵심 키워드를 자연스럽게 포함 → 이미 반영됨
- 모바일 대응·HTTPS·빠른 속도 → 정적 사이트라 기본 충족

## 2. 구글 서치콘솔

주소: https://search.google.com/search-console

1. **속성 추가**
   - 커스텀 도메인: "도메인" 속성(DNS TXT 레코드) 권장
   - GitHub Pages 서브경로: "URL 접두어" 속성으로 배포 주소 입력
2. **소유확인** — "HTML 태그" 방식: `<meta name="google-site-verification" content="...">` 값을 페이지 `<head>` 주석을 풀어 입력(네이버와 동일한 위치)
3. **Sitemaps**에 `sitemap.xml` 제출
4. **URL 검사** 도구로 주요 페이지 색인 요청

### 구글 리치 결과 메모
- 이 사이트는 `WebApplication` + `BreadcrumbList` JSON-LD를 사용합니다(검색결과에 무료 유틸리티/경로 표시에 유리).
- **FAQ·HowTo 구조화 데이터는 일부러 넣지 않았습니다.** 구글이 2023년 이후 해당 리치 결과를 폐지/제한했기 때문입니다(콘텐츠로서 FAQ는 본문에 그대로 있습니다).
- 실제 사용자 평점이 없으므로 `aggregateRating`을 임의로 넣지 않았습니다(스팸 정책 위반 방지).

## 3. 추가 노출 채널 (선택)

- **카카오톡 공유 미리보기**: OG 태그(`og:title/description/image`)가 이미 있습니다. `assets/img/og-default.png`를 매력적으로 다듬으면 클릭률이 올라갑니다.
- **다음(Daum)**: 별도 등록 도구가 약하지만, 네이버/구글 색인이 되면 일부 노출됩니다.
- **블로그·커뮤니티 백링크**: "PDF 합치기 무료" 등으로 글을 쓰며 링크를 걸면 초기 노출에 도움이 됩니다.

- [ ] 네이버 소유확인 + 사이트맵 제출 + 웹페이지 수집요청 완료

- [ ] `https://<도메인>/robots.txt` 와 `/sitemap.xml`이 브라우저에서 열린다
- [ ] [구글 리치 결과 테스트](https://search.google.com/test/rich-results)에서 JSON-LD 통과
